## 🤖 MCP란 무엇인가?

### 정의

**MCP (Model Context Protocol)**는 Claude AI와 외부 도구/시스템을 연결하는 표준 프로토콜입니다.

- **역할**: Claude가 로컬 파일시스템, 명령어, API 등에 접근할 수 있도록 중개
- **목적**: AI가 단순한 텍스트 대화가 아닌 실제 개발 작업을 수행하도록 지원
- **구조**: Claude ↔ MCP Server ↔ 외부 도구/시스템

## 🔧 Git 상태 읽기에 사용되는 도구

### 1️⃣ 직접 사용한 도구

| 도구명 | 타입 | 설명 | 사용 방법 |
|--------|------|------|---------|
| **Bash** | 쉘 명령 실행 | git 명령어 직접 실행 | `git status` |
| **Glob** | 파일 검색 | 파일 패턴 매칭 | `git` 관련 파일 찾기 |
| **Grep** | 텍스트 검색 | git 정보 텍스트 검색 | git log/diff 분석 |
| **Read** | 파일 읽기 | 로컬 파일 읽기 | `.git/config`, `HEAD` 파일 읽기 |

## 📋 각 MCP/도구의 역할

### Bash (쉘 명령 실행)

**주요 git 명령어**:
```bash
git status          # 상태 확인
git log             # 커밋 이력
git diff            # 변경사항 비교
git add <file>      # 파일 스테이징
git commit -m "msg" # 커밋 생성
git branch          # 브랜치 관리
git checkout        # 브랜치 전환
git push            # 원격 저장소 푸시
git pull            # 원격 저장소 풀
```

**사용 예**:
```bash
cd "e:\work\dev_dir\claude" && git status
```

---

### Glob (파일 패턴 검색)

**git 관련 파일 찾기**:
```
패턴: .git/**/*
패턴: **.gitignore
패턴: **.git/config
```

**장점**: 빠른 파일 검색, 경로 패턴 지원

---

### Grep (텍스트 검색)

**커밋 메시지 검색**:
```bash
grep -n "feature" .git/logs/HEAD
```

**변경 파일 검색**:
```bash
git log --name-only | grep "\.js$"
```

**장점**: 정규식 지원, 여러 파일 동시 검색

---

### Read (파일 읽기)

**git 설정 읽기**:
```
.git/config       # 저장소 설정
.gitignore        # 무시할 파일 목록
HEAD              # 현재 브랜치 정보
```

**장점**: 직접 파일 내용 확인 가능

---

### Edit / Write (파일 수정)

**git과 무관하지만 개발 작업에 필수**:
- 코드 편집
- 커밋 메시지 작성
- `.gitignore` 파일 수정

## 💡 Best Practices

### Git 상태 빠르게 확인
```bash
# 가장 빠른 방법
git status
```
→ **Bash** 직접 사용

### 파일 변경사항 상세 보기
```bash
git diff HEAD~1..HEAD
git log -p -1
```
→ **Bash** + **Grep**

### 브랜치 전환 후 파일 수정
```bash
git checkout -b feature/new-feature  # Bash
# 파일 수정
Edit / Write                         # Edit/Write
git add .                           # Bash
git commit -m "message"             # Bash
```
→ **Bash** + **Edit** + **Write** 조합
